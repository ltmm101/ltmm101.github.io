Title, Summary and Team Members

Title: Position Based Fluid Simulation

Members: Leah Tom and Patrick Lutz

Summary: Particle based fluid simulation using position based dynamics (i.e. updating particle positions directly rather than accumulating forces on each particle and then updating accelerations), density constraints, artificial pressure, and vorticity confinement, based on the paper “Position Based Fluids” by Miles Macklin and Matthias Muller.

Background

Modelling the behavior of fluids and then rendering it have been recognized as challenges for several reasons: the need to track the changing surface of the fluid, enforcing constraints like incompressibility and other behaviours of liquids like tensile instability. Though there are algorithms for modelling the behavior of liquids over time with minuscule spheres, most of these tend to be computationally expensive. In particular, the Smoothed Particle Hydrodynamics (SPH) model being used currently is unstable in cases where a particle does not have enough neighbours to make accurate density estimates. In these cases, the remedy then becomes reducing the size of each time step at the expense of increased computation.
The Position Based Dynamics (PSD) framework handles these edge cases more gracefully by clamping pressures to be non-negative and adding additional pressure terms in order to maintain the values of particle density at convenient values in specific cases. These, however, do come at the expense of maintaining accurate values of incompressibility, the tradeoff being improved computation speeds. Resources

We will be aiming to create a short animation. We plan to have approximately 3 frames per second. We plan to have our pictures rendered and then stitch them together using Windows Movie Maker. Since we are drawing relatively basic shapes like spheres, we plan to use OpenGL to draw them onto screen. We will probably just be coding up the basic classes like a water droplet and functions and algorithms that check if collision has occurred between different water droplets (i.e. using a hash grid to keep track of this). If we have time, we will try to render the surface of the water too or try to improve the perception of depth in the movie with a pathtracer.

Goals and Deliverables

Our goal is to create a short animation of a ball dropping into a small pool of water and to accurately model the splash created.

In order to evaluate how successful we have been in implementing our project, we will need to measure the time taken for the algorithm we implement to run and compare it to the time taken by other more standard algorithms like the Smoothed Particle Hydrodynamics. One possible way of doing this might be to use the animation utility in Blender that creates animations of liquids based on the SPH algorithm. Another alternative might just be to code it up ourselves. With the alternative version implemented, we will do a direct comparison of how much time it took for us to render our images in one implementation versus the other.

What we plan to deliver:

We plan to deliver an animation that has around 5 frames per second and is a realistic representation of water's behavior when a ball is dropped into it. We will achieve this by accounting for the following properties of water: incompressibility, tensile instability and vorticity confinement and viscosity.

What we hope to deliver:

We hope that we will also have time to code up an alternative implementation of our animation using a different algorithm to compare the performance of our algorithm against. We also hope to be able to improve upon the existing PSD algorithm and to add in our own optimizations, if we can find any.

If we are ahead of schedule:

We hope to be able to add in a pathtracer to improve the depth perception with lighting effects. We also hope to have enough time to be able to render our body of liquid. This will require additional research on implementing algorithms that will keep track of the surface of the liquid.
